# Asset And Image Brief

## 核心原则

视频图和公众号贴图不是一回事。

公众号贴图的目标是：

- 独立可读
- 一张图讲完整

视频图的目标是：

- 作为模板里的一个证据片段
- 配合切换去推进理解

## 视频素材要拆成什么

不要只产“整张总结海报”。

一条视频至少拆成这些素材角色：

1. `headline-base`
   - 顶部结论层用
2. `proof-01`
   - 第一张证据图
3. `proof-02`
   - 第二张证据图
4. `proof-03`
   - 第三张证据图
5. `footer-copy`
   - 底部解释层文案

## 三类常用证据图

### 1. 列表证据图

适合：

- 合集
- 推荐项
- 多个 skill 并列

要求：

- 每张图只放少量条目
- 给后续条目留出推进空间

### 2. 仓库/界面证据图

适合：

- GitHub 项目
- 工具首页
- 平台界面

要求：

- 截图主体清晰
- 不要过度裁切
- 重点区域可放大，但不要全屏模糊

### 3. 结构/机制证据图

适合：

- 系统解释
- 概念类比
- 流程说明

要求：

- 一图只证明一个关系
- 不要全图堆满说明字

## 生图或改图时的约束

- 不要默认做成公众号配图海报
- 不要在一张图里塞满“标题 + 解释 + 总结 + CTA”
- 中部证据图只需要证明一个点
- 允许留白
- 允许上下分区
- 允许把正文信息拆成 2 到 3 张图

## 适合的视频图提示词方向

不是：

- “完整知识海报”
- “公众号图文配图”

而是：

- “知识短视频中部证据卡”
- “视频号信息板式证据窗素材”
- “用于短视频切换展示的结构图 / 仓库卡 / 对比卡”

## 文案层面的约束

视频专用素材里，单张图尽量只承担一种功能：

- 讲结论
- 讲对象
- 讲证据
- 讲结果

不要一张图四种都讲。

## 推荐交付命名

- `headline_base.png`
- `proof_01.png`
- `proof_02.png`
- `proof_03.png`
- `footer_copy.txt`

如果是列表型，可用：

- `list_item_01.png`
- `list_item_02.png`
- `list_item_03.png`

## 反例

下面这些很容易做废：

- 把公众号贴图整张直接塞进视频
- 一张图上已经有很多字，还再叠大标题
- 中部证据图既像海报又像文章摘要
