# Reference Patterns

这份文件把 3 条参考视频抽成 3 个可复用模式。

## Pattern A: `list-build`

适用：

- 清单
- 推荐合集
- 多个 skill 一次性推荐

结构：

1. 顶部主标题固定
2. 标题下副条固定
3. 列表项逐条出现
4. 每条列表项是：
   - 名称
   - 一句话说明
5. 底部提示条固定

真实节奏感来自：

- 名称逐条流式打出
- 说明逐条补完
- 页面内容不是“整页切换”，而是“向下生长”

适合时长：

- `10s - 15s`

不适合：

- 单个工具深度介绍
- 复杂系统解释

## Pattern B: `hook-proof-switch`

适用：

- 单个工具
- 单个 skill
- 单个项目
- 一个明确问题的解决方案

结构：

1. 顶部爆点标题固定
2. 中部证据窗切换
3. 底部高亮说明固定

中部证据窗可切：

- GitHub 仓库首页
- 功能结构页
- 原理页
- 数据表
- 漏洞/结果页

真实节奏感来自：

- 中部证据窗快速换图
- 换图时有轻微模糊、缩放、推入
- 顶部标题不频繁重排

适合时长：

- `5s - 8s`

不适合：

- 长列表合集
- 需要 5 个以上论点推进的内容

## Pattern C: `concept-diagram-switch`

适用：

- 机制解释
- 系统关系
- 概念翻译
- 框架介绍

结构：

1. 顶部一句好记的概念判断
2. 中部图窗轮流展示关系图、曲线图、流程图
3. 底部说明卡固定

真实节奏感来自：

- 同一概念下切换不同证明图
- 让观众从“听懂一句话”到“看到证据”

适合时长：

- `5s - 8s`

不适合：

- 需要观众阅读大量段落

## 三种模式的共同骨架

不管是哪一种，骨架都一样：

1. 顶部留人
2. 中部证明
3. 底部翻译

变化主要发生在中部。

## 选择规则

如果内容里有 3 个以上并列对象，优先 `list-build`。

如果内容里只有 1 个核心对象，但有 2 到 3 个可证明它的证据，优先 `hook-proof-switch`。

如果内容难点在“理解概念”，优先 `concept-diagram-switch`。
